Q&A

  • What are some differences between web1, web2 and web3?

    1. During web1 all websites were static, which means that only the
      webmaster could modify the website. In web2 and web3, the websites became
      dynamic, which means that also the Internet surfers could start contributing to
      the websites (which could now support Online Social Networks).

    2. web3 supports a Semantic Web, which makes use of semantic languages
      in order to return better results to search queries. For example, the query "What side effects
      others have for Acamol drug?" would get pretty good results thanks to the Semantic Web, while
      in web1 you would get worse results becuase it web1 doesn't support a Semantic Web.
      web2 Does support semantic search to a certain extent, but the web it supports is not
      considered "Semantic".

    3. web3 Will make extensive use of AI algorithms (machine learning, NLP, etc), unlike web1
      and web2), in order to discover new trends on the web, improve semantic search, etc.


  • Can you explain these differences with a "Use Case" example?

    I'll describe the use case of a user in Tel Aviv that wants to search for a good place to eat in the evening:

    • On web1:
      1. The user gets to the search site.
      2. Types in "Good restaurants in Tel Aviv".
      3. Clicks the first link that appears (Tel-Aviv.com), not being sure it's the best site for this.
      4. In the homepage of the site there is no info about restaurants, but there is a link in the menu.
        The user clicks this link ("Restaurants")
      5. The user has to manually scroll through the page, comparing the restaurant's address with
        His own address and decide by himself if the restaurant is good enough (there's no information
        about the restaurant's score, because the site is static and user's cannot give scores) and close enough.
      6. The user finds a restaurant that suits his needs. The only information he gets from the site is
        The restaurant's phone number and the menu, so he has to call to ask if there's enough tables at that time.

    • On web3:
      1. The user gets to the search site.
      2. Types in "Good restaurant near me".
      3. The search results are the best restaurants that are close to the user. The user clicks the first result
        because that restaurant is very near, it has a great score and also serves his favorite food.
      4. The user is presented with the option to reserve a table, which he chooses. After clicking, the table
        is automatically reserved without the user having to call and manually reserve it.

    As you can see, in web3 made use of AI and semantics to understand the meaning of the query, and thanks to the websites
    being dynamic could sort the search results according to the score given by many other users.
